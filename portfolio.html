01 핵심 요약

맥락 없는 AI 분석은, 속도만 빠른 착각이다

AI에게 마케팅 데이터를 주면 빠르게 분석해 준다. 그러나 '왜'를 모른 채 내리는 판단은, 분석이 아니라 오판의 자동화다. 인게임 이벤트로 ROAS가 급등했을 때 "예산을 증액하세요"라고 할 것이고, 플랫폼 정책 문제로 지출이 급감했을 때 "매체를 이동하세요"라고 할 것이다.

5개 데이터소스를 종합 수집하고, 맥락 기반으로 판단하고, 가드레일로 검증한 뒤, 매체 API를 통해 직접 실행까지 하는 마케팅 AI 에이전트를 설계하고 구축했다.

항목Before (현재)After (에이전트 도입)

이상치 대응 발견까지 평균 6시간, 수동 조정 1~2시간 실시간 탐지, 승인 즉시 API 자동 조정

오판 리스크 맥락 없는 분석 → 이벤트 기간 예산 증액 등 가드레일이 6개 항목 자동 체크 후 판단

핵심은 리소스 절감이 아니다. 판단과 실행 사이의 시간차를 제거하여 예산 손실을 막는 것이다. 본 시스템은 설계 단계가 아닌, 실제 구축 및 운영 테스트를 완료한 상태다.

02 문제 정의

왜 지금의 AI 마케팅 분석은 위험한가

ROAS가 올랐다고 예산을 늘리면? — 이벤트 끝나면 전부 손실이다

마케팅 데이터의 변동에는 항상 이유가 있다. 인게임 이벤트, 매체 세팅 변경, 플랫폼 정책 이슈, SDK 장애 등 — 숫자 뒤에 숨은 원인을 모른 채 분석하면 필연적으로 오판한다.

실제 상황LLM의 오판실제 원인
인게임 페이백 이벤트로 ROAS 급등 "성과 개선! 예산 증액!" 이벤트 종료 시 원상복귀. 증액하면 예산 낭비
SDK 장애로 iOS 이벤트 수집 지연 "iOS 설치 급감! CPI 폭등!" SDK 지연으로 인한 일시적 집계 오류

공통점: 데이터에는 '무엇이' 변했는지만 있고, '왜' 변했는지는 없다.

데이터는 넘친다 — 그러나 담당자는 한두 명이다

AppsFlyer, 매체 대시보드, 내부 BI — 데이터 소스는 넘치지만 추출, 해석, 인사이트 도출은 1~2명의 담당자에게 집중된다. AI 분석이 늘어날수록 검증할 여력은 줄고, 잘못된 분석이 걸러지지 않고 의사결정에 반영될 위험이 커진다.

AI만 믿으면 판단을 그르치고, 사람에게만 의존하면 한계가 있다.
AI가 맥락을 이해하되, 최종 판단은 사람이 하는 구조가 필요하다.

03 솔루션

5개 데이터소스를 종합 판단하고, 가드레일로 검증하고, 직접 실행한다

3.1 종합 판단 — 단일 소스가 아닌, 모든 데이터를 끌어온다

MMP 데이터만으로는 "왜?"를 알 수 없다. 에이전트는 활용 가능한 모든 외부 데이터를 종합하여 판단한다.

📊 MMP

AppsFlyer, Adjust 등에서 1차 성과 데이터를 수집. CPI, ROAS, 코호트 등 핵심 KPI의 기준점.

📢 매체 API

Google Ads, Meta Ads에서 캠페인 변동 이력을 확인. 예산/입찰가/타겟팅 변경 사항을 자동 추적.

📱 스토어 콘솔

App Store, Play Store에서 앱 업데이트, 점검 이력을 확인. 설치수 변동의 외부 요인 파악.

🔍 센서타워

경쟁사 카테고리 시장 동향 확인. 경쟁사가 코스트를 늘려 경쟁이 심화됐는지 파악.

💾 내부 BI

1st Party Data와 매칭하여 유저 퀄리티까지 추적. 설치 뿐 아니라 매출 기여도까지 판단.

하루 시나리오: 에이전트가 판단하는 과정

08:30
KPI 브리핑 자동 발송
MMP + 매체 + 내부 BI에서 어제 데이터를 수집 → 통합 CPI/CPA/ROAS 요약 → 두레이 + 이메일로 자동 전달. 출근길에 바로 확인.
09:00
이상치 알림 — "iOS CPI 40% 급등"
MMP에서 이상 감지 → 스토어 콘솔 확인: "SDK 업데이트 지연" → 센서타워 확인: "경쟁사 공격적 매수 없음" → 원인 자동 매칭: 기술 이슈. 신뢰도 HIGH
10:00
예산 변경 요청 → 가드레일 작동
담당자가 예산 증액 요청 → Decision Guard 6항목 자동 체크 → 2개 FAIL(이벤트 기간, 데이터 부족) → "이벤트 종료 후 재평가 권장" 신뢰도 LOW
14:00
주간 리포트 자동 생성
5개 데이터소스를 종합 분석 → What Happened + Why(내부BI 매출 퀄리티 + 센서타워 시장 변동) + What To Do(액션 제안) 포함 리포트.
17:00
크리에이티브 알림 + 매체 자동 조정
"소재 A그룹 CPI 30% 상승, Off 제안" → 담당자 승인 → Meta Ads API로 직접 Off 실행. 동시에 Google Ads 예산 5% 재배분 실행.

같은 데이터, 다른 판단:

MMP 데이터만 보는 AI

"ROAS 42% 급등! 예산 증액하세요!"

5개 소스 종합 판단 + 가드레일

"ROAS 42%는 페이백 이벤트 영향(Context Layer). 센서타워 시장 변동 없음. 내부 BI 실질 매출 기여 20%. 이벤트 종료(3/7) 후 재평가."
신뢰도 LOW 이벤트 기간 겹침 Decision Guard 2/6 FAIL

3.2 AI 판단 검증 장치 (가드레일)

AI의 모든 판단에 신뢰도 등급을 강제 표기하고, 고위험 의사결정은 6항목 자동 체크를 통과해야 한다.

🎲

신뢰도 등급

모든 판단에 자신감 수준 강제 표기

HIGH MEDIUM LOW
☑️

Decision Guard

예산 증감/매체 ON-OFF 등 고위험 권고 전 6항목 자동 체크

👤

Human-in-the-Loop

최종 의사결정은 반드시 사람이 승인

Decision Guard 6개 항목 상세 보기
  1. 데이터 충분성 — 최근 7일 이상의 데이터에 기반하는가?
  2. 이벤트 분리 — 프로모션/이벤트 기간이 포함되어 있지 않은가?
  3. ML 안정화 — 매체 설정 변경 후 ML 재학습 기간(7일)이 경과했는가?
  4. 외부 영향 배제 — 정책 변경/SDK 장애 등 외부 요인이 없는가?
  5. 이상치 배제 — 이상치 플래그된 날짜가 포함되어 있지 않은가?
  6. 크로스 매체 일관성 — 다른 매체와 비교했을 때 일관된 트렌드인가?

하나라도 FAIL이면 해당 권고는 "의사결정 보류 — 추가 데이터 필요" 상태로 출력.

3.3 분석을 넘어, 실행까지

에이전트는 분석에서 끝나지 않는다. 매체 API를 통해 직접 세팅을 변경하고, 크리에이티브를 관리한다.

🎯

매체 세팅 자동 조정

예산 증감, 입찰가 조정, 캠페인 ON/OFF를 승인 후 API로 직접 실행. 담당자는 판단만, 실행은 에이전트가.

🎨

크리에이티브 자동 관리

소재별 성과 스코어링 → 저효율 소재 자동 Off, Top 패턴 도출. 소재 교체 타이밍까지 제안.

📊

리포트 자동 생성

일간/주간/월간 3종 리포트를 자동 생성하여 두레이 + 이메일로 전달.

왜 실행까지 해야 하는가?
마케터 리소스 절감만이 목적이 아니다. 판단과 실행 사이의 시간차가 성과 손실의 핵심 원인이다.
이상치를 발견하고도 매체 대시보드에 들어가 수동으로 조정하는 데 1~2시간이 걸린다. 그 사이 예산은 계속 소진된다. 에이전트가 실행까지 담당하면 발견 즉시 대응이 가능해진다.
